## 추천 실행안

1. 실행 전 17계정을 순차 프리워밍한다.
   - 로그인 세션 확인
   - 보안문자/추가인증 계정 제외
   - 실패 계정은 스케줄에서 즉시 제거

2. 댓글 생성은 먼저 완료하거나, 최소 1~2라운드 분량을 선생성한 뒤 실행한다.
   - DeepSeek 생성 예상: 40~90분
   - 생성 결과는 글별 8개, 첫 6글자 중복/시작어 반복/길이 검증

3. 실행은 기본 큐 worker를 무작정 높이지 말고, 계정별 병렬 러너로 진행한다.
   - 동시 활성 계정: 8~12개부터 시작
   - 계정별 최소 간격: 8분
   - 같은 글 다음 댓글 간격: 최소 60분 이상 유지
   - 전체 글을 1회차, 2회차 방식으로 순환

4. 첫 30분은 파일럿으로 40~60댓글만 처리한다.
   - 실패율 10% 이상이면 중단
   - 로그인 실패/캡차/댓글 입력창 없음/검증 실패를 유형별 집계
   - 이상 없으면 나머지 계속 진행

## 구현상 주의점

- `task_global` worker concurrency만 올리면 같은 큐의 모든 작업이 같이 빨라져 다른 운영 작업과 섞일 수 있다.
- 같은 계정의 동시 작업은 account lock으로 막히지만, 다른 계정이 같은 글에 동시에 접근하는 것은 별도 제한이 필요하다.
- 댓글 작성 함수가 `forceFreshLogin: true`라서 대량 병렬 시 로그인/보안확인 부하가 커질 수 있다.
- 스케줄 실행 중 세션이 풀리면 해당 계정만 중단하고 다른 계정은 계속 가는 구조가 필요하다.
- 완료 후 네이버 실제 댓글 수와 DB 기록을 다시 대조해야 한다.
