# 카페 캠페인 스케줄 큐 추가

키워드 파일 기반으로 광고/일상 글을 시간대별로 스케줄링하여 큐에 추가합니다.

## 작업 흐름

### 1단계: 입력 확인

사용자에게 아래 정보를 확인합니다:

- **대상 카페**: 기본 4개 (쇼핑지름신 + 샤넬오픈런 + 건강한노후준비 + 건강관리소). 빼달라고 하지 않는 한 항상 4개 포함.
- **카페별 글 수**: 기본값 — 쇼핑 광고10+일상5, 샤넬 일상5+일상광고5, 건강노후 광고3+일상9, 건강관리 광고3+일상9
- **샤넬 일상광고(daily-ad)**: 일상 카테고리(일상샤반사) + 일상 원고 스타일로 작성. 댓글 차단됨. 나중에 modify-posts로 광고로 전환 예정.
- **샤넬오픈런 게시판 규칙 (필수)**: 샤넬 카페 글은 타입과 무관하게 **반드시 `_ 일상샤반사 📆` 게시판**만 사용. `_ 오늘의 쇼핑 🛒` 포함 다른 게시판 사용 금지.
- **광고 타사:자사 비율**: 7:3 (쇼핑/건강 광고 키워드에만 적용, 샤넬은 일상광고라 해당 없음)
- **마감 시간**: 기본 오후 10시 (22:00)

### 2단계: 키워드 선정

1. **광고 키워드 소스 (구글 시트 우선)**:
   - **1순위: 구글 시트** — 모든 카페 광고 키워드의 주 소스
   - Spreadsheet ID: `1zU8Jr2qH_T1JkMoi6SHgWbeeQBLMbLan8wLQ3jSlmhI`
   - 탭: `카페_관련 키워드` (gid: `1588523146`)
   - 읽는 범위: `E:H`
   - 컬럼 해석:
     - `E`: 키워드
     - `F`: 카페명 — 비어 있으면 자사, 값이 있으면 타사
     - `G`: 노출구좌
     - `H`: 노출 유/무
   - **다양하게 랜덤 추출**: 특정 주제(선물, 흑염소 등)에 편중되지 않게 다양한 카테고리에서 골고루 선택
   - 관련 임시 조회 스크립트:
     - `scripts/fetch-kw-temp.ts` — 원본 E:H 조회
     - `scripts/filter-kw-temp.ts` — `used-keywords.txt` 제외 후 미사용 자사/타사 후보 조회
   - **2순위: `scripts/keywords/ad-keywords.txt`** — 시트에 키워드가 부족할 때 보조 소스
   - **3순위: 기존 사용 키워드 재사용** (`used-keywords.txt`에서 재활용) — 1·2순위 모두 소진 시
   - **쇼핑지름신 광고 구성**: 타사옹호(competitor-advocacy) 7개 + 자사(own) 3개 = 10개
     - 타사옹호: 사용자가 직접 제공하는 제품 키워드 (keywordType: "competitor-advocacy")
     - 자사: 흑염소 관련 키워드 (keywordType: "own")
2. **사용된 키워드 제외**: `scripts/keywords/used-keywords.txt`에 있는 키워드 제외
3. **중복 제거 + 셔플**: 남은 키워드에서 랜덤 셔플로 필요한 수만큼 선택
4. **일상 키워드 생성**: 시간대에 맞는 자연스러운 일상 키워드 직접 생성
   - **필수**: 반드시 실제 고유명사(브랜드, 드라마 제목, 제품명, 매장명 등)를 포함할 것
   - **금지**: "넷플릭스 보면서", "인스타 구경", "유튜브 리뷰" 같은 추상적/포괄적 표현 단독 사용 금지
   - **올바른 예시**: "월간남친 보면서 간식 먹는 월요일 밤", "샤넬 26SS 시갈백 인스타 보는데 실물 궁금해요", "쿠팡 로켓배송 장바구니 정리하다가 3만원 넘김"
   - **나쁜 예시**: "넷플릭스 보면서 간식 먹는 월요일 밤", "퇴근하고 샤넬 매장 인스타 구경하는 중"
   - 웹 검색으로 현재 유행하는 드라마/제품/이벤트를 확인 후 키워드에 반영

### 3단계: 시간 배정

일상 키워드는 내용에 맞는 시간대에 배치:

| 키워드 유형 | 적합 시간대 |
|------------|-----------|
| 점심/메뉴 관련 | 11:00~13:00 |
| 오후 간식/졸림 | 13:00~15:00 |
| 카페/산책/날씨 | 시간 무관 (낮) |
| 퇴근/저녁반찬 | 17:00~19:00 |
| 드라마/야식 | 19:00~21:00 |

광고 키워드는 시간 무관, 현재~마감까지 균등 분배.

**계정 간격 규칙**: 같은 계정은 최소 1시간 이상 간격 유지.

**카페 내 계정 연속 금지 (필수)**: 같은 카페 게시판에서 시간순 정렬 시, 같은 계정의 글이 연속으로 보이면 안 됨. 같은 계정의 글 사이에 반드시 **다른 계정의 글이 최소 2개 이상** 끼어 있어야 함.
- 예: 쇼핑지름신에 A가 13:30 글 → 다른 계정 B, C 글 → A가 16:00 글 (OK)
- 금지: A가 13:30 글 → A가 14:00 글 (같은 카페에서 연속)
- 검증: SCHEDULE 배열을 카페별로 그룹핑한 뒤, 시간순 정렬하여 같은 accountId가 연속인지 체크할 것

**키워드 겹침 최소화 (필수)**: 같은 카페 내에서 동일/유사 메인 키워드의 글이 연속 배치되지 않도록 함. 흑염소 관련 키워드끼리, 선물 관련 키워드끼리 등 유사 주제가 몰리지 않게 섞을 것. 불가피한 경우에만 허용.

**제목 패턴 중복 방지 (필수)**: 생성된 글 제목이 같은 카페 게시판 목록에서 비슷하게 보이면 안됨. 특히 건강카페 흑염소 광고에서 "흑염소진액 먹어본 후기", "흑염소진액 선물 고민이에요", "흑염소진액 추천 부탁드려요" 같은 패턴이 반복되면 봇 티가 남.
- **금지 패턴**: 같은 카페에 같은 주제·동일 어미 제목이 연속/근접하게 배치되는 것
- **검증 시점**: 큐 잡을 추가하기 전에 이미 발행된 오늘 글의 title + 큐 대기 중인 잡의 subject를 같이 묶어서, 첫 12자 stem 단위로 그룹핑해 중복 stem이 없는지 확인할 것
- **수정 방법**: 중복 stem 발견 시 해당 잡의 subject를 다른 화법(질문/후기/일상/매장 방문/가격 비교 등)으로 갈아끼우고 rawContent의 `[제목]` 섹션도 같이 수정. (예: "흑염소진액 먹어본 후기 궁금해요" → "청궁흑염소 한달 먹어본 솔직 후기")
- 실행: `bull:*:delayed`의 `data.subject` 와 `data.rawContent` 값을 redis hset으로 패치

**카페/타입 혼합 규칙 (필수)**: 카페(쇼핑/샤넬)와 타입(광고/일상)을 모두 뒤죽박죽 섞어서 배치. 다음 패턴 모두 금지:
- 한 카페 글을 먼저 몰아서 배치 후 다른 카페 배치 (쇼핑 다 → 샤넬 다)
- 광고를 모두 먼저 배치 후 일상 배치 (광고 다 → 일상 다)
- 올바른 예시: 쇼핑ad → 샤넬ad → 쇼핑ad → 쇼핑daily → 샤넬daily → 쇼핑ad → 샤넬ad → 쇼핑daily → ...

**계정당 배분 규칙 (필수)**:

| 카페 | 계정당 배분 | 5계정 합계 |
|------|-----------|----------|
| 쇼핑지름신 | 광고 2개 + 일상 1개 | 광고10 + 일상5 = 15개 |
| 샤넬오픈런 | 광고 1개 + 일상 1개 | 광고5 + 일상5 = 10개 |

- 각 계정(olgdmp9921, yenalk, eytkgy5500, uqgidh2690, 4giccokx)에 동일하게 배분
- SCHEDULE 배열에서 계정별 타입 분포 검증 후 확인 요청할 것
- 일상 키워드는 해당 계정의 배정 시간대에 맞게 할당 (점심 → 13~14시 슬롯 계정에게)

### 4단계: 사용자 확인

아래 형식의 표로 전체 스케줄을 보여주고 확인받습니다:

```
## 카페명

| 키워드 | 카테고리 | 타입 | 계정 | 시간 |
|--------|---------|------|------|------|
```

수정 요청이 있으면 반영 후 재확인.

### 5단계: 큐 추가 실행

확인 받으면 `scripts/run-schedule.ts`의 SCHEDULE 배열을 업데이트하고 실행합니다.
**실행 전 반드시 LOGIN_ID 기본값이 `21lab`인지 확인할 것.**

```bash
npx tsx --env-file=.env.local scripts/run-schedule.ts
```

### 6단계: 사용된 키워드 기록

실행 완료 후, 이번에 사용한 **광고 키워드만** `scripts/keywords/used-keywords.txt` 파일 맨 아래에 추가합니다.

```bash
# 예시: 사용한 광고 키워드 목록을 한 줄씩 append
cat >> scripts/keywords/used-keywords.txt << 'EOF'
출산선물추천
임산부선물추천
60대여자선물
EOF
```

- 일상 키워드는 기록하지 않음
- 추가 후 사용자에게 추가된 키워드 목록을 알려줌

### 7단계: 카페별 광고 발행 키워드 보고

사용된 광고 키워드 기록까지 끝나면, 사용자에게 **카페별 광고 발행 키워드**를 줄바꿈 목록으로 별도 보고합니다.

- 최소 필수 보고 대상: `샤넬오픈런`, `쇼핑지름신(쇼지)`
- 다른 카페가 함께 실행된 경우, 동일한 형식으로 이어서 추가 보고 가능
- 형식은 아래처럼 제목 1줄 + 빈 줄 + 키워드 목록으로 맞춤

```text
샤넬 광고 발행키워드

키워드
키워드

쇼지 광고 발행키워드

키워드
키워드
```

## 댓글 동작

`run-schedule.ts`는 글 생성 시 `parseViralResponse`로 rawContent의 `[댓글]` 섹션을 파싱하여 `viralComments`를 PostJobData에 포함시킴.
글 발행 후 commenter 3~5개 계정이 3분 간격으로 자연스럽게 댓글을 달게 됨 (viral-batch와 동일 로직).
